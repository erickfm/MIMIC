## What Changes with Intra-Frame Attention (hybrid16)

```
WITH FLAT ENCODER (current):
  55 features вҶ’ concat вҶ’ MLP вҶ’ 1 vector per frame
  Sequence transformer does ALL reasoning (temporal + cross-feature)

WITH HYBRID16 ENCODER (old, unstable):
  55 features вҶ’ group into 16 tokens вҶ’ 2 layers of attention вҶ’ pool to 1 vector
  The per-frame attention pre-computes cross-feature relationships
  Sequence transformer only needs to do temporal reasoning

  Per-frame attention example:
    Token "self_action=JUMPSQUAT" attends to token "self_analog=(0.5, -0.8)"
    вҶ’ Produces a representation that already encodes "wavedash angle detected"
    вҶ’ Sequence transformer just needs to reason about timing

  The problem: 2 extra attention layers per frame Г— 180 frames = 360 attention
  operations in the backward pass, creating deep gradient paths that explode.
```
