Inside One Attention Head

One head processes all 180 frames. Each frame gets a Query, Key, and Value.
Q
what am I looking for?
K
what do I contain?
V
what info to extract?
Computing output for frame 47:
score(47→1) = Q47 · K1 / √96   ← how relevant is frame 1?
score(47→2) = Q47 · K2 / √96
⋮
score(47→46) = Q47 · K46 / √96   ← previous frame
score(47→47) = Q47 · K47 / √96   ← self
score(47→48) = −∞   ← causal mask: can't see the future
⋮
score(47→180) = −∞
weights = softmax(scores)
e.g. [0.01, 0.01, …, 0.40, 0.30, …]
      frame 1   frame 2      frame 46   frame 47
output47 = w1·V1 + w2·V2 + … + w47·V47
A weighted average of all past frames' values. The weights are learned — the model decides what's relevant.
What each head might specialize in:
Head 1: "Previous frame"
Temporal continuity — what just happened?
Head 2: "Same action"
Find past frames with matching action state
Head 3: "Opponent movement"
Track where the opponent was and is going
Head 4: "Recent context"
Smooth attention over last ~10 frames
Head 5: "Feature extraction"
Attend to self (frame 47→47) to mix features
Head 6: "State transitions"
Find when actions changed — transition timing
Head 7: "Spacing"
Track relative positioning over time
Head 8: "Combo/tech patterns"
Recognize multi-frame input sequences

Flat vs Hybrid16 Encoder

Stable
Flat Encoder (current)
55 features → embed each → concat → 2-layer MLP → 768-d

No per-frame attention. All cross-feature reasoning happens in the sequence transformer.

The model uses temporal attention slots to understand within-frame relationships (e.g., frame 47 attends to frame 46, which has near-identical content).

gnorm = 0.38 at 1M+ steps
Unstable
Hybrid16 Encoder (old)
55 features → group into 16 tokens → 2 layers of self-attention → pool → 768-d

Pre-computes cross-feature relationships within each frame. “JUMPSQUAT + diagonal stick = wavedash” encoded before the sequence transformer sees it.

2 extra attention layers × 180 frames = deep gradient paths that eventually explode.

gnorm → 509,047 at 2.4M steps (ctx=60)
gnorm → 126,100 at 410k steps (ctx=180)
What we lose with flat: The inductive bias that features within a frame should interact. The flat MLP can learn fixed linear combinations (weighted sums of features), but can't do conditional reasoning like “attend to stick angle only when action is JUMPSQUAT.” The sequence transformer can still learn this, but it must use attention slots that could otherwise do temporal reasoning.

What we gain: Complete gradient stability, matching HAL's proven architecture. HAL achieves 95% win rate vs CPU with flat concat at 20M params.
