Transformer Backbone × 6

Pre-Norm Block (modern components)
RMSNorm
Replaces LayerNorm. No mean subtraction, single learnable γ. Faster and equally effective.
x · rsqrt(mean(x²) + ε) · γ
Relative Position Attention
Shaw et al. skew-based relative position bias. Learnable Er matrix per layer. Better than RoPE for frame sequences.
Att = softmax((QK¹ + skew(QEr¹)) / √d)
Grouped Query Attention
8 query heads share 2 KV heads (4:1 ratio). Reduces KV parameters while maintaining query expressiveness.
Q: 8 heads × 64d  |  K,V: 2 heads × 64d (repeated 4×)
SwiGLU FFN
Gated activation replaces GELU. SiLU gate modulates a parallel up-projection. ~1-2% better than standard FFN.
SiLU(xWgate) ⊙ xWup → Wdown
Per block:
  x = x + RelPosAttn(RMSNorm(x))  — residual + attention
  x = x + SwiGLU(RMSNorm(x))  — residual + feedforward
Output: (B, 256, 512) after 6 blocks + final RMSNorm

Why keep RelPos? RoPE (used by LLaMA, GPT-4, etc.) was tested and performed slightly worse on this task. Frame sequences have different positional structure than language tokens — the learnable relative position matrix captures frame-to-frame timing patterns that fixed rotary frequencies miss. All other modern components (RMSNorm, SwiGLU, GQA) improved or matched performance.
GQA at this scale: With only 2 KV heads sharing across 8 query heads, KV parameters drop significantly. The freed parameter budget goes to a slightly larger SwiGLU FFN (2050 base vs 2048), keeping total params at ~20M. At inference, GQA also reduces KV-cache memory by 4×.

Training Configuration

Optimizer: AdamW (β=0.9, 0.999; wd=0.01; grad clip=1.0)
LR schedule: CosineAnnealingLR, no warmup, ηmin=1e-6
Learning rate: 3e-4
Batch size: 512 effective (64 × 8 grad_accum on 1 GPU, or 64 × 8 GPUs)
Precision: FP32 (no AMP)
Dropout: 0.2 throughout
Loss: Plain cross-entropy on all 4 heads (no focal loss, no weighting)
Sequence: 256 frames (~4.3 seconds at 60 fps)
Init: N(0, 0.02) weights, zero bias, residual scaling 0.02/√(2×6)
